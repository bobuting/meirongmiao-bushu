**端点 1: POST /image-projects/:projectId/step3/photos/generate-batch**

功能: LLM 规划模特图方案（8-12 张），创建记录并逐张生成

请求体:
```typescript
{
  outfitSummary?: string;
  characterDescription?: string;
}
```

处理流程:
1. requireUser + requireOwnerProject 校验
2. 验证项目已完成 Step 2: 查询项目数据，确认 selectedOutfitPlanId 和 selectedModelId 存在（参考 step2-handlers 中获取项目数据的方式）
3. 获取 Step 1 搭配方案: ctx.repos.outfitPlans.findById(project.selectedOutfitPlanId)
4. 获取 Step 2 定妆图: ctx.repos.characterFiveViews.findActiveByCharacterId(project.selectedModelId)，提取 imageUrl 数组作为参考图
5. 构建 LLM userPrompt，包含: 搭配方案摘要、角色描述、要求输出 JSON 格式数组（每项: poseLabel, bgLabel, posePrompt, bgPrompt）
6. 调用 getPromptContent(PROMPT_CODE_STEP3_PLAN, { userPrompt })
7. 调用 resolveRouteProviderWithFallback(ctx, ["script_generation"]) 获取 LLM Provider
8. 调用 requestLlmPlainText 获取 JSON 规划结果
9. 解析 JSON（使用 extractJsonObject），生成 8-12 张 ModelPhoto 规划
10. 批量创建 ModelPhoto 记录（status='pending', isSelected=false, order=1..N）: 使用 ctx.repos.modelPhotos.bulkCreate()
11. 获取图片生成 Provider: resolveRouteProviderWithFallback(ctx, ["image_to_image", "text_to_image"])
12. 逐张同步生成（8-12 张约 2-4 分钟，可接受）:
    - 更新 status='generating'
    - 构建生成 prompt（结合 posePrompt + bgPrompt）
    - 如果有 Step 2 定妆参考图，使用 mode='image_to_image' + images=[定妆图URL]
    - 否则使用 mode='text_to_image'
    - 调用 requestJimengImageUrl 生成图片 URL
    - 调用 persistImageSourceToStorage(ctx, imageUrl, `image-projects/${projectId}/model-photos`, { persistRemote: true })
    - 更新 imageUrl, status='success'
    - 如果失败: 更新 status='failed', errorMessage（单张失败不影响其他张）
13. 返回 { photos: ModelPhoto[], plan: { total, poses: [...], backgrounds: [...] } }
