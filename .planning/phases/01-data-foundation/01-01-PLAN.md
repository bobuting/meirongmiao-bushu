# Plan 01-01: 类型定义 + 合约契约

## Objective

建立图片项目 4 步流水线所需的全部 TypeScript 类型定义和合约契约。

## Must Haves

1. `ModelPhoto`、`PageSection`、`SectionVersion` 类型存在于 `src/contracts/types.ts`
2. 图片项目 step1-4 的 snapshot 类型存在于 `project-step-snapshot.ts`
3. `IMAGE_PROJECT_MAX_STEP = 4`，clamp/filter 逻辑覆盖 step1-4
4. `ProjectFlowStep` 类型支持图片项目的 step1-4

## Tasks

### Task 1: 定义核心数据类型

在 `src/contracts/types.ts` 中添加以下类型：

```typescript
// 模特图（Step 3）
export interface ModelPhoto {
  id: string;
  projectId: string;
  imageUrl: string | null;
  poseLabel: string;          // 如 "正面全身站立"
  bgLabel: string;            // 如 "纯色渐变棚拍"
  isSelected: boolean;        // 是否选入素材池
  status: 'pending' | 'generating' | 'success' | 'failed';
  errorMessage: string | null;
  order: number;
  createdAt: number;
  updatedAt: number;
}

// 电商详情页区块（Step 4）
export interface PageSection {
  id: string;
  projectId: string;
  sectionKey: string;           // 项目内唯一标识
  type: 'HERO' | 'SELLING_POINTS' | 'SCENARIO' | 'DETAIL' | 'TRUST' | 'SIZE' | 'CUSTOM';
  title: string;
  goal: string;
  copy: string;
  visualPrompt: string;         // 视觉提示词
  order: number;
  status: 'idle' | 'generating' | 'success' | 'failed';
  currentImageAssetId: string | null;
  editableData: Record<string, unknown>;
  createdAt: number;
  updatedAt: number;
}

// 区块版本历史（Step 4）
export interface SectionVersion {
  id: string;
  sectionId: string;
  projectId: string;
  versionNumber: number;
  promptSnapshot: Record<string, unknown>;
  copySnapshot: Record<string, unknown>;
  imageAssetId: string | null;
  isActive: boolean;
  createdAt: number;
}
```

### Task 2: 更新图片项目 step snapshot 类型

修改 `src/contracts/project-step-snapshot.ts`，新增图片项目专用的 step1-4 snapshot：

```typescript
// 图片项目 Step 1: 服装搭配
export interface ImageStep1Snapshot {
  uploadedAssetIds: string[];       // 上传的服装图片 ID
  outfitAnalysisResult: string | null;  // AI 搭配分析结果
  selectedOutfitPlanId: string | null;
  createdAt: number | null;
}

// 图片项目 Step 2: 角色定妆
export interface ImageStep2Snapshot {
  characterPreviewIds: string[];    // 生成的定妆图 ID
  selectedModelId: string | null;
  styledViewIds: string[];          // 确认的定妆图
  confirmedModel: boolean;
  createdAt: number | null;
}

// 图片项目 Step 3: 模特图生成
export interface ImageStep3Snapshot {
  modelPhotoIds: string[];          // 生成的模特图 ID
  selectedPhotoIds: string[];       // 用户选中的模特图
  generating: boolean;
  createdAt: number | null;
}

// 图片项目 Step 4: 电商详情页
export interface ImageStep4Snapshot {
  sectionIds: string[];             // 页面区块 ID
  heroSectionCount: number;
  detailSectionCount: number;
  generatingAll: boolean;
  createdAt: number | null;
}
```

更新 `ProjectStepSnapshotEnvelope`，新增 `imageSteps` 字段：

```typescript
interface ImageProjectStepSnapshotEnvelope {
  projectId: string;
  updatedAt: number;
  lastVisitedStep: 1 | 2 | 3 | 4;
  steps: {
    step1: ImageStep1Snapshot;
    step2: ImageStep2Snapshot;
    step3: ImageStep3Snapshot;
    step4: ImageStep4Snapshot;
  };
}
```

同时更新 `createEmptyProjectStepSnapshot()` 函数返回新的 `imageSteps` 字段。
更新 `isProjectStepSnapshotEnvelope()` 验证函数，增加对 `imageSteps` 的校验。
注意：不修改视频项目的 `steps` 结构，保持向后兼容。

### Task 3: 更新项目流程类型和常量

修改 `src/contracts/project-last-step.ts`：
- 保持 `ProjectFlowStep = 1 | 2 | 3 | 4 | 5` 不变（视频项目用）
- 新增 `ImageProjectFlowStep = 1 | 2 | 3 | 4`（图片项目用）

修改 `apps/web/pages/project-flow/projectFlowKind.ts`：
- `IMAGE_PROJECT_MAX_STEP` 从 `2` 改为 `4`
- `clampProjectFlowStepForKind` 的 clamp 上限改为 4
- `isProjectFlowStepAllowed` 允许 image 类型的 step 1-4
- `filterWorkflowStepsByProjectKind` 返回图片项目 4 步
- `layoutNavigationController.ts` 中更新图片项目的导航标签

### Task 4: 更新前端导航标签

修改 `apps/web/components/layout/layoutNavigationController.ts`：
- 图片项目导航步骤标签从 2 步改为 4 步：
  1. "服装搭配"
  2. "角色定妆"
  3. "模特图生成"
  4. "电商详情页"
