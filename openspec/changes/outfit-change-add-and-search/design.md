## Context

换装项目 Step 2（选择服装）和 Step 3（选择角色）当前仅支持从已有库中选择。用户需要：
1. 在流程内快速新增服饰/角色
2. 通过搜索快速定位目标项

**技术背景：**
- Step 2 使用 `realGarmentAssetsApi.listGarmentAssets()` 获取服饰列表
- Step 3 使用 `backendApi.listLibraryCharacters()` 获取角色列表
- 已有服饰上传 API（`realGarmentAssetsApi.createGarmentAsset`）
- 已有角色创建 API（通过 `backendApi`）

## Goals / Non-Goals

**Goals:**
- Step 2 添加"新增服饰"按钮 + 上传弹窗
- Step 3 添加"新增角色"按钮 + 创建弹窗
- Step 2/3 添加搜索输入框，支持按名称过滤
- 新增成功后自动刷新列表并显示新项

**Non-Goals:**
- 不修改后端 API
- 不添加高级搜索（如分类、标签过滤）
- 不修改服饰/角色管理页

## Decisions

### 1. 新增按钮位置

**决策：** 放在搜索框右侧或列表上方

```
[搜索输入框] [新增按钮]
```

**理由：** 用户先搜索，找不到再新增，符合操作流程。

### 2. 弹窗实现方式

**决策：** 使用内联弹窗（Modal）组件

**实现：**
- Step 2：复用或新建服饰上传弹窗（上传图片 + 输入名称）
- Step 3：复用或新建角色创建弹窗（输入名称 + 选择基础图）

**备选方案：** 跳转到专门页面 → 不选，因为中断流程

### 3. 搜索实现方式

**决策：** 前端即时搜索（不调用后端 API）

**实现：**
- 搜索状态：`useState` 存储 `searchQuery`
- 过滤逻辑：`items.filter(item => item.name.includes(searchQuery))`
- 搜索后分页：在过滤结果上进行分页

**理由：**
- 数据已在前端（分页实现时已获取全部）
- 即时响应更流畅
- 不增加后端负担

### 4. 搜索输入框样式

**决策：** Material Icons `search` + 输入框 + `close` 清除按钮

```
🔍 [输入框____________] ✕
```

- 响应式：移动端输入框宽度自适应
- 清除按钮：点击清空搜索并恢复全部列表

## Risks / Trade-offs

### 风险 1：新增弹窗可能复杂

**风险：** 角色创建弹窗可能涉及多步操作（选择基础图、设置属性等）

**缓解：** 先实现简单版本（仅输入名称 + 上传图片），后续迭代

### 风险 2：搜索与分页的交互

**风险：** 搜索结果数量可能少于每页显示数量

**缓解：** 搜索时禁用或调整分页；或搜索结果独立分页

## Open Questions

1. **新增服饰弹窗是否需要分类选择？** 建议先简化，仅名称 + 图片
2. **新增角色弹窗是否需要五视图上传？** 建议先简化，仅名称 + 单张图
