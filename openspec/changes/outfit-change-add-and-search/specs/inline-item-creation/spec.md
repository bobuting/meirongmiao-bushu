## ADDED Requirements

### Requirement: Step 2 支持流程内新增服饰

换装项目 Step 2 页面 SHALL 提供新增服饰按钮，允许用户在流程内直接上传新服饰，无需跳转其他页面。

#### Scenario: 点击新增服饰按钮打开弹窗

- **WHEN** 用户点击"新增服饰"按钮
- **THEN** 系统打开服饰上传弹窗
- **AND** 弹窗包含图片上传区域和名称输入框

#### Scenario: 上传服饰成功后自动刷新列表

- **WHEN** 用户在弹窗中上传图片并输入名称，点击确认
- **THEN** 系统调用服饰创建 API
- **AND** 成功后自动刷新服饰列表
- **AND** 新增的服饰出现在列表中供选择

### Requirement: Step 3 支持流程内新增角色

换装项目 Step 3 页面 SHALL 提供新增角色按钮，允许用户在流程内直接创建新角色，无需跳转其他页面。

#### Scenario: 点击新增角色按钮打开弹窗

- **WHEN** 用户点击"新增角色"按钮
- **THEN** 系统打开角色创建弹窗
- **AND** 弹窗包含名称输入框和基础图上传区域

#### Scenario: 创建角色成功后自动刷新列表

- **WHEN** 用户在弹窗中输入名称并上传基础图，点击确认
- **THEN** 系统调用角色创建 API
- **AND** 成功后自动刷新角色列表
- **AND** 新增的角色出现在列表中供选择

### Requirement: 新增按钮位置和样式

新增按钮 SHALL 位于搜索框右侧，使用 Material Icons `add` 图标，响应式样式。

#### Scenario: 移动端显示图标

- **WHEN** 用户在移动端查看 Step 2/3 页面
- **THEN** 新增按钮仅显示 `add` 图标

#### Scenario: 桌面端显示图标和文字

- **WHEN** 用户在桌面端查看 Step 2/3 页面
- **THEN** 新增按钮显示 `add` 图标和"新增"文字
