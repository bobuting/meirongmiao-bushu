## shot_description 生成规则（重要）

`shot_description` 是每个镜头的**观众视角画面描述**。把所有 shot_description 按顺序连起来读，观众应该能看懂发生了什么故事，不需要任何额外解释。

### 核心原则：聚焦动作和场景，不描写服饰细节

shot_description 只描述**观众在屏幕上实际看到的东西**。**服饰和角色的视觉外观由参考图（五视图+平铺图）保证**，文本不需要重复描述服饰的颜色、材质、款式等细节。文本聚焦于参考图无法传递的信息：动作、表情、镜头运动、场景、情绪变化。

### 写法规则

| 规则 | 说明 |
|------|------|
| **只写可见** | 人物动作、表情、位置、场景物件——观众能在屏幕上直接看到的东西 |
| **按时间顺序** | 镜头内事件按发生顺序写，先发生什么再发生什么，逻辑清晰 |
| **因果关系** | 如果有因果关系（A导致B），必须明确写出，不能省略 |
| **人物指代明确** | 用"主角"、"穿搭配1的角色"区分人物，不用"她"、"另一个"。多角色时用简短标识区分 |

### 禁止的写法

| 禁止类型 | 错误示例 | 为什么错 | 正确写法 |
|----------|----------|----------|----------|
| **情绪评价** | "空气紧绷" | 观众看不到"空气" | "两人面对面站着不动，谁也不看谁" |
| **情绪标签** | "余韵悠长"、"温暖释然" | 这是感受不是画面 | "女生嘴角微微翘起，低头看着自己的鞋" |
| **拍摄术语** | "现实虚化"、"柔光叠影" | 观众不关心怎么拍的 | "画面变模糊，两个身影重叠在一起走" |
| **特效描述** | "倒带效果"、"定格帧" | 描述的是技术不是画面 | "两人托举的动作快速倒放，回到站立的姿势" |
| **内心活动** | "她开始想象"、"陷入回忆" | 观众看不到别人的内心 | 直接描述想象/回忆的画面内容 |
| **抽象概括** | "找到属于自己的节奏" | 没有画面感 | "两人的脚步声开始同步，踩在同一个节拍上" |
| **详细服饰描写** | "米色棉质卫衣与温暖光线融为一体" | 服饰外观由参考图保证，文本描述冗余且可能冲突 | "坐在洒满阳光的窗边，专注地看书" |

### 生成公式

```
shot_description = [谁（简短锚点）] + [在做什么] + [在哪/什么环境] + [结果或反应]
```

**示例**：

- ✅ "主角退了半步，搭档顺势轻托她的手肘，两人开始一起左右晃动"
- ✅ "镜头扫过镜子上贴的便签纸，上面写着'今天也要开心'，背景里两个人在笑"
- ❌ "穿深灰外套的女生退了半步，穿针织衫的搭档顺势轻托她的手肘，两人开始一起左右晃动"（服饰细节多余）
- ❌ "两人意外合拍后相视而笑，镜面便签点题，留下接纳笨拙的温暖余韵"

### 连贯性检查（关键）

生成所有分镜后，**必须做连贯性检查**：

1. **按顺序读完所有 shot_description**——能看懂一个完整的故事吗？
2. **相邻分镜有因果关系**——前一个镜头的结果，是后一个镜头的起因吗？
3. **没有跳跃**——人物从"站着"突然变成"坐在地上"，中间发生了什么？
4. **结尾有交代**——最后一个镜头停在什么画面上？观众能感受到"结束"了吗？
5. **每个人物都有结局**——出场了的人物，最后在做什么？

### 与其他字段的关系

| 字段 | shot_description 必须与之一致 |
|------|------------------------------|
| `subjects[].action` | 动作描述一致 |
| `subjects[].expression` | 表情描述一致 |
| `subjects[].clothing` | 仅保持锚点标识一致（如"搭配1"），不写服饰细节 |
| `visual.scene` | 场景描述一致 |
| `visual.lighting` | 光线氛围一致 |
| `audio.ambient_sound` | 环境音基调一致 |
