# 多主体交互规则

> 当画面中出现多个角色/主体时，提供空间关系描述模板和互动因果链。
> **前提**：仅当分镜 JSON 的 subjects 数组包含多个主体时才适用。空间关系应从分镜的视觉分析数据中推导，不凭空创造原文没有的位置信息。

## 空间位置描述模板

多主体场景应从分镜数据中提取并明确每个主体的空间位置关系：

```
[主体A] 位于画面 [位置]（左/右/中/前景/背景），面向 [方向]（镜头/另一主体/画外）
[主体B] 位于画面 [位置]，与主体A的相对距离 [远/中/近]，[面对/背对/侧对] 主体A
```

### 示例
```
女主角位于画面右侧前景，面对镜头左侧方向；男主角位于画面左侧中景，面对女主角方向，两人之间约一臂距离，形成面对面对话构图
```

## 互动因果链

描述角色间的互动时，必须建立因果关系：

```
[角色A的动作] → [角色B的反应] → [环境/氛围变化]
```

### 示例
```
A伸出手触碰B的肩膀 → B转头看向A，表情从惊讶变为微笑 → 两人之间的空间距离缩短，氛围从紧张变为温暖
```

## 焦点分配策略

多主体场景中焦点有主次之分：

| 主体角色 | 描述权重 | 细节程度 |
|----------|---------|---------|
| 主焦点 | 60% | 完整的面部表情、服饰细节、动作描述 |
| 次焦点 | 30% | 关键表情和动作，服饰简略 |
| 背景/群像 | 10% | 仅轮廓和姿态，不描述面部细节 |

## 遮挡处理规则

当主体之间或主体与环境存在遮挡时：

1. **只描述可见部分**：被遮挡的区域不描述细节（如"人物右半身被柱子遮挡"→ 不描述右侧服饰）
2. **利用遮挡创造层次**：前景遮挡物增加画面深度（如"前景中另一人的肩膀虚化，后景主体清晰"）
3. **标注遮挡关系**：在 prompt 中明确谁遮挡谁（如"A partially blocking B from camera view"）
