# 服饰平铺图 Logo 保持优化分析

> 分析时间：2026-04-13  
> 接口：`POST /garment-assets/generate-flat-lay`  
> 提示词模板：`garment_flat_lay_generation`

---

## 🎯 问题背景

在使用 AI 图片生成服务进行服饰平铺图生成时，服饰上的 **品牌 logo、商标、刺绣图案** 等细节会出现变形、消失或被替换的情况。

---

## 📊 问题根因分析

### 技术原因

#### 1. AI 模型的生成机制
- 图生图模型（Image-to-Image）在处理时会对原图进行"重建"
- 模型倾向于"优化"细节，导致小面积图案被模糊或忽略
- 训练数据中的版权保护机制会主动"避免"精确复制品牌 logo

#### 2. 提示词约束不足
- 当前提示词 `garment_flat_lay_generation` 中**没有明确提及 logo 保持**
- 第 9 条只提到"细节纹理和图案"，但未强调 logo/商标的特殊重要性
- 缺少具体的 logo 保持技术指导

#### 3. 图片生成 API 参数限制
- 当前使用的 `requestJimengImageUrl()` 函数没有专门的 logo 保持参数
- 即梦 API 等第三方服务的图生图接口可能不支持"局部保护"功能

---

## 🔍 当前提示词约束分析

### 现有约束（第 9-12 条）

```
9. 衣服的细节纹理和图案必须高度还原原图，还原度一致性强，不允许模糊、丢失或变形
10. 颜色必须与原图完全一致，零色差，严禁偏色、褪色或过饱和
11. 保留原始服饰的材质细节、缝线、纽扣、拉链等微小元素
12. 对于原图中未直接展示、不清晰或无法确认的部分细节，仅进行保守填充，严禁随意创造、编造或臆测不存在的图案和纹理
```

### 问题分析

- ✅ 第 9 条提到了"图案"，但**未明确优先级**
- ❌ 第 12 条反而会**误导模型**：如果 logo 稍微不清晰，模型会"保守填充"，导致 logo 被简化或忽略
- ❌ 没有**具体的技术指导**，如"像素级还原"、"边缘锐化"等

---

## 💡 优化方案

### 方案一：提示词优化（低成本，快速实施）

#### 优化策略
在 System Prompt 中新增专门的 logo 保持约束，明确优先级和技术要求。

#### 具体优化内容

```markdown
## Logo 与品牌标识保持（最高优先级）

服饰上的品牌 logo、商标、刺绣、印花、标签等品牌标识是核心商业价值所在，必须像素级精确还原：

1. **识别与定位**
   - 自动识别服饰上的所有品牌标识（胸标、袖标、领标、吊牌等）
   - 记录每个标识的精确位置、大小、形状、颜色、字体

2. **像素级还原要求**
   - Logo 必须与原图 100% 一致，包括：边缘轮廓、线条粗细、字体样式、颜色饱和度
   - 严禁对 logo 进行模糊、简化、变形、重新设计或替换
   - Logo 的背景、阴影、边缘锯齿必须保留原样

3. **技术实现指导**
   - 使用高精度边缘检测保留 logo 轮廓
   - 对 logo 区域进行局部锐化和色彩校正
   - 如果原图 logo 不清晰，保持原样输出，严禁"优化"或"增强"

4. **多视角融合规则**
   - 如果正面图有 logo，背面图必须同步保持
   - 如果多张参考图的 logo 细节不一致，选择最清晰、最完整的版本

5. **禁止行为**
   - 禁止删除、遮挡、替换任何品牌标识
   - 禁止更改 logo 的颜色、字体、比例、位置
   - 禁止将品牌 logo "通用化"或"去品牌化"
```

#### 插入位置
- 放在 System Prompt 的 **"忽略无关内容"** 之前，**"生成要求"** 之前
- 确保优先级高于普通细节约束

---

### 方案二：图片生成 API 参数优化（中成本）

#### 优化策略
检查即梦 API 等服务商是否支持以下高级参数：

1. **局部保护（Inpainting Mask）**
   - 指定图片的某些区域（如 logo 区域）保持不变
   - 需要前端配合标注 logo 位置

2. **局部重绘强度（Region Strength）**
   - 对 logo 区域使用极低的重绘强度（strength: 0.1）
   - 对其他区域使用正常强度（strength: 0.7）

3. **图像增强（Post-processing）**
   - 在生成后对 logo 区域进行锐化和色彩校正
   - 使用 OpenCV 等库进行局部处理

#### 技术实现示例

```typescript
// 伪代码示例
const generatedResult = await requestJimengImageUrl(routeResult.provider, prompt, {
  mode: "image_to_image",
  images: imageUrls,
  ratio: "9:16",
  // 新增参数（需要确认 API 是否支持）
  preserveRegions: logoBoundingBoxes, // 需保护的区域坐标
  regionStrengths: { // 不同区域的重绘强度
    logo: 0.1,       // logo 区域几乎不重绘
    clothing: 0.7,   // 服饰正常重绘
    background: 0.9  // 背景完全重绘
  }
});
```

### 方案三：两阶段生成流程（高成本，最佳效果）

#### 流程设计

```
阶段 1: Logo 检测与提取
├── 调用视觉模型（如 GPT-4o-Vision）分析原图
├── 识别所有 logo/商标位置
├── 提取 logo 区域并单独保存
└── 生成 logo mask（遮罩）

阶段 2: 服饰平铺图生成
├── 使用当前流程生成平铺图（包含正面/背面）
└── 应用 logo 保护规则

阶段 3: Logo 精确合成
├── 将提取的 logo 精确贴回对应位置
├── 进行边缘融合和色彩校正
└── 输出最终结果
```

#### 技术栈
- **Logo 检测**: GPT-4o Vision / Gemini Vision / YOLO
- **图像合成**: OpenCV / Sharp（项目已有）
- **边缘融合**: Poisson Blending / Laplacian Pyramid
