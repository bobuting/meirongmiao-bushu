# JSON 格式化工具设计文档

> 日期：2026-05-12
> 状态：已批准

## 1. 概述

为开发调试场景设计一个独立工具页面，支持 JSON 格式化美化、节点折叠/展开、搜索过滤、实时编辑等功能，能处理 1MB-10MB 的大型 JSON。

## 2. 需求

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 格式化美化 | 将压缩 JSON 转为缩进格式 | P0 |
| 压缩 | 去除空白，输出最小化 JSON | P0 |
| 节点折叠/展开 | 对象/数组节点可折叠，显示 `...N 项` 占位 | P0 |
| 全部展开/折叠 | 一键操作所有节点 | P0 |
| 搜索过滤 | 搜索 key/value，高亮匹配行，显示匹配数 | P0 |
| 实时编辑 | 左侧 textarea 编辑，右侧实时预览 | P0 |
| 复制 | 复制格式化结果到剪贴板 | P1 |
| 下载 | 下载为 .json 文件 | P1 |
| 文件拖拽 | 支持拖拽 .json 文件到输入区 | P1 |
| 右键菜单 | 复制值、复制路径 | P2 |

### 2.2 非功能需求

- **性能**：1MB-10MB JSON 可正常使用（虚拟滚动 + Web Worker 解析）
- **大型数组/对象默认折叠**：超过 100 个子项时自动折叠
- **输入防抖**：300ms，避免频繁重渲染

## 3. 方案设计

### 3.1 整体布局

双栏分体式布局，可拖拽分隔条调整比例（默认 50:50）：

```
┌─────────────────────────────────────────────────────────┐
│  {} JSON Formatter  │ 格式化 压缩 复制 下载 │ 展开 折叠 │ 🔍搜索 │
├──────────────────────┼──────────────────────────────────┤
│  ● 输入    2.1 KB   │  ● 预览    42 节点 · 深度 4      │
│  1 │ {              │  ▾ {                           42  │
│  2 │   "name": ...  │    ▾ "projectName": "neirong..."  │
│  ...                │    ▾ "config": ...                │
├──────────────────────┴──────────────────────────────────┤
│  ● JSON 有效  │ 节点: 42  │ 深度: 4  │         2.1 KB │
└─────────────────────────────────────────────────────────┘
```

### 3.2 工具栏

| 按钮 | 快捷键 | 说明 |
|------|--------|------|
| 格式化 | `⌘⇧F` | 美化缩进 |
| 压缩 | `⌘S` | 去除空白 |
| 复制 | — | 复制到剪贴板 |
| 下载 | — | 下载 .json 文件 |
| 展开全部 | — | 展开所有折叠节点 |
| 折叠全部 | — | 折叠所有对象/数组 |
| 搜索 | `⌘F` | 搜索 key/value |

### 3.3 左侧面板 — 输入编辑器

- textarea 带行号显示
- 支持拖拽 .json 文件
- 输入时实时解析（300ms 防抖）
- 显示字符数/文件大小

### 3.4 右侧面板 — 树形预览

- **语法着色**：key 蓝色 `#0066cc`、string 绿色 `#1a7f37`、number 橙色 `#d97e10`、boolean 紫色 `#7c3aed`、null 灰色
- **折叠交互**：点击 ▾ 箭头或 `...N 项` 占位文本切换折叠
- **搜索高亮**：匹配行黄色边框 + 当前匹配加粗
- **虚拟滚动**：只渲染可视区域节点（大型 JSON 性能保障）
- **右键菜单**：复制值、复制 JSON 路径（如 `$.config.api.timeout`）

### 3.5 状态栏

- JSON 有效性指示（绿/红圆点 + 文字）
- 节点数、最大深度
- 文件大小

## 5. 性能策略

### 5.1 虚拟滚动（大型 JSON 核心）

右侧树形视图只渲染可视区域的 DOM 节点：

- 计算所有节点的扁平列表和高度
- 根据 scrollTop 计算可见范围
- 只渲染可见节点 + 上下缓冲区（各 5 行）
- 超出部分用空 div 占位保持滚动条正确

### 5.2 Web Worker 解析

JSON.parse 在 Web Worker 中执行，避免阻塞 UI 线程：

- 主线程发送原始字符串到 Worker
- Worker 解析后返回结构化数据
- 大于 500KB 的 JSON 强制走 Worker
- Worker 超时 5s 后提示用户

### 5.3 默认折叠策略

- 数组/对象子项 > 100 个时默认折叠
- 递归深度 > 8 时默认折叠子节点

### 5.4 输入防抖

300ms 防抖，避免用户快速输入时频繁重渲染。

## 6. 文件结构

```
apps/web/pages/json-formatter/
  └── JsonFormatter.tsx    # 单文件独立页面，参照 image-viewer 模式
```

路由在 `App.tsx` 中添加：
```
<Route path="/json-formatter" element={<LazyPage><JsonFormatter /></LazyPage>} />
```

无需 RequireAuth，公开访问。

## 7. 组件拆分（JsonFormatter.tsx 内部）

| 组件/函数 | 职责 |
|-----------|------|
| `JsonFormatter` | 主页面组件，管理状态 |
| `EditorPanel` | 左侧输入编辑器 |
| `TreePanel` | 右侧树形预览 |
| `VirtualTree` | 虚拟滚动树形视图 |
| `TreeNode` | 单个树节点渲染 |
| `Toolbar` | 顶部工具栏 |
| `StatusBar` | 底部状态栏 |
| `SearchBar` | 搜索输入框 |
| `useJsonParser` | 解析 + 统计 hook |
| `useVirtualScroll` | 虚拟滚动 hook |
| `useCollapsible` | 折叠状态管理 hook |

## 8. 交互细节

- 拖拽分隔条：调整左右面板比例（25%-75% 范围）
- 空状态：未输入时显示引导提示
- 错误状态：解析失败时显示错误信息
- Toast 提示：操作反馈（格式化成功/失败等）
- 右键菜单：在树形视图上右键弹出
